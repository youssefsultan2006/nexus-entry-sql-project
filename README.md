# 🛡️ Nexus Entry – Gym Entry System

A SQL database project designed to manage gym memberships, track entries, and simulate a real-world fitness facility management system.

## 🚀 Key Features
- Member Management: Track membership tiers, expiration dates, and personal data.
- Access Control: Log entry and exit times to monitor gym capacity.
- Reporting & Analytics: SQL queries to identify active members, peak hours, and operational insights.

## 🛠️ Technical Implementation
- Designed an EER (Enhanced Entity-Relationship) Diagram to model relationships between members, staff, and facilities.
- Built a fully normalized relational database structure using MySQL Workbench.
- Applied SQL concepts including:
  - Joins
  - Aggregations
  - Subqueries
  - Data filtering and retrieval

## 💻 Tools & Technologies
- MySQL Server
- MySQL Workbench
- SQL

## 📂 Project Structure
- `/Diagrams` → EER Diagram & database structure
- `/Scripts` → Table creation and data insertion scripts
- `/Queries` → SQL queries for reporting and business logic

## 📈 Project Goal
The objective of this project was to simulate how real-world database systems support operational efficiency, data management, and business decision-making.
